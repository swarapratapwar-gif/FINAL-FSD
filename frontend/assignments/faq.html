Frequently Asked Questions

Find answers to common questions about the EduAchieve portal.

General Questions

Who can use EduAchieve? ▼
EduAchieve is exclusively available to students, faculty, and external examiners associated with Pimpri Chinchwad College of Engineering (PCCOE). Registration requires a valid @pccoe.org email address.
Are my uploaded documents public? ▼
Basic project details like abstract and tech stack are visible to all visitors. However, downloadable assets like PDFs and PPTs are only fully accessible based on the permissions set by the admin and the platform's current access policies.
How do I reset my password? ▼
If you are logged in, you can change your password from the Settings page. If you have forgotten your password, please contact the administrator as the automated password reset feature is currently under development.
Can I update my project after submission? ▼
Yes, the original author of the project can edit the project details, including re-uploading documents, from the "My Profile" or "Project Details" page.
What is the maximum file size for uploads? ▼
The maximum file size for PDF research papers and PPT presentations is 50MB. Cover images must be under 5MB and in PNG or JPG format.

Technical Questions

What technologies are used to build this platform? ▼
EduAchieve is built using a modern stack: Node.js and Express for the backend, Vanilla JavaScript, HTML5, and CSS3 for the frontend, and JWT for secure authentication.
How is authentication handled? ▼
We use JSON Web Tokens (JWT). Upon successful login, a token is issued and stored in your browser's localStorage. This token is then sent with every API request requiring authorization via the Authorization header.
Where is the data stored? ▼
Currently, the platform uses a local JSON file-based database (db.json) to store user and project metadata, making it highly portable and easy to run locally without external database dependencies.
Is the platform responsive? ▼
Yes, the entire platform is built with a mobile-first approach using CSS Flexbox and Grid, ensuring it works seamlessly across desktop, tablet, and mobile devices.
How does the dark mode work? ▼
Dark mode uses CSS Custom Properties (variables). When the toggle is clicked, a .dark-mode class is added to the body tag, which overrides the light theme variables with dark theme equivalents. The preference is saved in localStorage.
